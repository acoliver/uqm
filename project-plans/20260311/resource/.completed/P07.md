## Implementation Summary

### Critical Fix Applied
**Location:** `rust/src/resource/ffi_bridge.rs`, lines 442-468

**Change:** SaveResourceIndex now skips entries without toString function instead of falling through to a fallback format.

**Before:**
```rust
let serialized = if desc.res_type == "STRING" {
    format!("{}:{}", desc.res_type, desc.fname)
} else if let Some(handlers) = state.dispatch.type_registry.lookup(&desc.res_type) {
    if let Some(to_string_fn) = handlers.to_string {
        // ... use toString
    } else {
        format!("{}:{}", desc.res_type, desc.fname)  // [ERROR] Fallback
    }
} else {
    format!("{}:{}", desc.res_type, desc.fname)  // [ERROR] Fallback
};
```

**After:**
```rust
// Look up handler by type_handler_key (not res_type, for UNKNOWNRES)
let handler = state.dispatch.type_registry.lookup(&desc.type_handler_key);
let to_string_fn = match handler.and_then(|h| h.to_string) {
    Some(f) => f,
    None => continue,  // [OK] Skip entries without toString
};

let serialized = if desc.res_type == "STRING" {
    format!("{}:{}", desc.res_type, desc.fname)
} else {
    // ... use toString (guaranteed to exist)
};
```

### Key Implementation Details

1. **Handler Lookup:** Uses `type_handler_key` instead of `res_type` to correctly handle UNKNOWNRES entries (where res_type != type_handler_key)

2. **Early Continue:** Entries without toString are skipped with `continue` before any serialization attempt

3. **Preserved Behavior:** Root filtering and strip_root functionality unchanged

4. **Testing Approach:** Since SaveResourceIndex calls UIO C functions not available in tests, filtering logic is tested via helper function at dispatch level with full-path testing documented as requiring UIO integration

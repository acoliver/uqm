## Requirements Verified

### REQ-STATE-001 â€” First-wins terminal reducer

- `TerminalMirror` uses `AtomicU8` with `NO_TERMINAL=255` sentinel.
- `try_set()` performs CAS from `NO_TERMINAL` to the terminal class â€” first
  call wins, later calls return `false`.
- Later errors are recorded as secondary and cannot turn failure into success.
- Tests: `first_wins_sets_once`, `success_is_first_wins_too`,
  `later_errors_are_secondary`, `arbitrary_terminal_sequence_first_wins`.

### REQ-STATE-002 â€” Synchronization failure model

- `TerminalCommand::terminal()` always returns `release_all=true,
  or_abort=true, stop=true`.
- `terminal_command()` on `RuntimeModel` returns `Some(TerminalCommand)` when
  terminal, `None` otherwise.
- Poison represented as terminal-only recovery (`PoisonedMutex` class).
- Unusable lock represented as atomic fallback (mirror works without mutex).
- Callback reentry represented as fail-closed without lock acquisition.
- Tests: `terminal_command_complete`, `terminal_command_always_release_all_or_abort_stop`.

### REQ-STATE-003 â€” Finalization ownership model + Lock-free mirrors

- Complete shell model: ABI entry (saturating) â†’ inactive fast path (neutral,
  no TLS/allocation/lock/external) â†’ active gate entry â†’ depth guard â†’
  terminal mirror check â†’ active shell count increment â†’ active.
- Fixed lock-free mirrors: terminal (`AtomicU8`), abort (`AtomicBool`),
  phase (`AtomicU8`), capture request (`AtomicU64`), owned keys (`OwnedKeyMirror`
  with `AtomicU64` mask + `[AtomicU8; 6]` values).
- Nested entry: `is_reentrant()` â†’ request_abort + release_all + terminal,
  without locking and never resuming scheduling.
- Tests: `inactive_shell_returns_fast_path`, `active_shell_returns_active`,
  `terminal_mirror_returns_conservative`, `reentry_returns_conservative`,
  `nested_entry_aborts_without_resume`, all sync_model tests.

### REQ-STATE-004 â€” Nonoverlapping lock-phase model

- Two-phase commit: `reserve_transition()` (checked state_version increment +
  reservation) â†’ unlock â†’ effects â†’ `commit_transition()` (version match

- RAII cancellation: `Reservation` Drop publishes `Cancelled` slot.
- Finalization: phase CAS â†’ clear capture â†’ check shells â†’ run_end once â†’
  finalized flag â†’ late callback blocked via `can_write()`.
- Lock-order instrumentation: `LockOrderTracker` rejects runtime-mutex overlap

- Tests: `reserve_and_commit_transition`, `stale_commit_rejected`,
  `finalize_completes_once`, `finalize_duplicate_run_end_rejected`,
  `late_callback_cannot_use_writer_after_finalize`,

## Key architectural decisions

- `cargo:rustc-link-arg-bin=uqm=-luqm_c` instead of `cargo:rustc-link-lib`:
  This is critical. `cargo:rustc-link-lib` would cause the staticlib to bundle
  C objects, creating duplicates when the C binary also links the same .o files.
  `cargo:rustc-link-arg-bin` only applies when linking the named binary.
- CFLAGS replicated from build.vars to ensure uqm.c/gameinp.c compile correctly
  with all necessary includes and defines.

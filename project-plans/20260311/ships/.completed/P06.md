### Handle Ownership Model

The implementation follows the C reference semantics:

1. `load_ship(species, MetadataOnly)` creates a temporary descriptor
2. Ship info and fleet data are copied into `MasterShipInfo`
3. Handle fields are zeroed in the descriptor to transfer ownership
4. Descriptor is dropped (safe because handles are zeroed)
5. `MasterShipInfo` owns the handles until catalog is freed
6. `free_master_ship_list()` frees the catalog-owned handles

Currently all resource IDs are 0, so handles are 0 (null), but the code is structurally correct for when real resources are loaded.

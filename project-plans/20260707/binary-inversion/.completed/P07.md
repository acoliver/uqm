# P07: E2E Boot Test — PASS

## Result

The Rust-owned binary (`rust/target/release/uqm`) successfully boots to the
main menu. This is the first time in the project's history that Rust owns
the process entry point.

## Verified at runtime

1. SDL initialization (v2.32.10)
2. Rust memory management initialized
3. Rust thread system initialized
4. Config/content directory mounting (Rust UIO)
5. Rust graphics driver (3 screen surfaces, 320x240)
6. Audio: Rust mixer + audio_heart backend
7. Resource loading (5 .rmp files)
8. LoadMaster: all 14 ships loaded successfully
9. **DoInput event pump**: [DoInput] log entries confirm single-threaded
   event pumping is working (TFB_ProcessEvents + ProcessUtilityKeys +
   TFB_FlushGraphics inside DoInput under #ifdef RUST_OWNS_MAIN)
10. Main menu reached: menu sound effects loaded, remix music loaded
    ("Control_the_Stars.ogg"), DoInput processing menu input
11. Audio mixer actively pumping (multiple mixer_pump_diag entries)
12. Process stable for 10+ seconds (killed by test harness, not crash)

## How to reproduce

```sh
cd rust
cargo build --bin uqm --release --features audio_heart
CONTENTDIR=$(pwd)/../sc2/content
./target/release/uqm --contentdir="$CONTENTDIR" -o
```

## Architecture confirmed

- Rust main() → uqm_c_do_init() → rust_game_loop() → game runs
- No separate game thread (StartThread eliminated)
- DoInput pumps SDL events in single-threaded mode
- Both Rust binary and C binary coexist (backward compatible)

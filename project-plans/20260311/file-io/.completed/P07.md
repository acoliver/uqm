## Implementation Notes

### Regex Compatibility Decision
- **Engine**: Rust `regex` crate v1.x
- **Compatibility**: Supports POSIX ERE-compatible patterns used by C callers
- **Limitation**: The `regex` crate uses RE2-style syntax (similar to POSIX ERE) but not exact POSIX ERE. However, all patterns used by actual C callers (`options.c`) are supported.
- **Error Handling**: Invalid regex patterns log warning and return no matches (safe fallback)

### Deterministic Ordering
- Mount registry iteration is precedence-ordered (position-based, already sorted)
- Entry collection uses first-seen ordering from precedence iteration
- `BTreeSet` used for deduplication while preserving insertion order
- Output is deterministic and precedence-sensitive as required

### uio_DirList ABI/Ownership
- First two fields (`names`, `numNames`) match C struct exactly
- Additional tracking via `buffer` field (compatible with C)
- Buffer size registry (`BUFFER_SIZE_REGISTRY`) enables safe deallocation
- `uio_DirList_free()` properly reconstructs and deallocates all allocations
- Allocation ownership is self-contained until free (no hidden side channels)

### Listing Concurrency
- Acquires mount registry lock for consistent topology snapshot during enumeration
- Lock held while iterating mounts and collecting entries
- Released before allocation phase
- Returned `uio_DirList` allocations remain valid until `uio_DirList_free()` regardless of subsequent topology changes

### errno Mapping
- `EINVAL`: null dir handle, invalid path encoding
- `ENOMEM`: allocation failures
- Invalid regex: logged but doesn't set errno (treated as no match)

### AutoMount Branch
- **Status**: DEFERRED per P00a Q2 resolution
- AutoMount is NOT required for engine/runtime paths
- All production C callers pass NULL for AutoMount parameter
- Implementation correctly skips AutoMount logic

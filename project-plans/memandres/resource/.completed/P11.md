## What Was Done
Combined with P09 since the logic was simple enough to implement directly:

### Core Implementation
- `put_string()` — creates STRING entry with both `fname` and `data_str` set to value
- `put_integer()` — creates INT32 entry with `data_num = value as u32`
- `put_boolean()` — creates BOOLEAN entry with `data_num = 0/1`
- `put_color()` — creates COLOR entry with packed RGBA in `data_num`
- `get_string()` — type-checked retrieval returning `Option<&str>`
- `get_integer()` — type-checked retrieval returning `Option<i32>`
- `get_boolean()` — type-checked retrieval returning `Option<bool>`
- `get_color()` — type-checked retrieval, unpacks RGBA tuple
- `serialize_entry()` — type-dispatched serialization using `serialize_color()`
- `save_resource_index()` — sorted output, root filtering, prefix stripping

### Design Decisions
- All Put functions unconditionally insert (no auto-create dance needed — HashMap handles it)
- All Get functions enforce strict type matching (return None on mismatch)
- Color format: `(r << 24) | (g << 16) | (b << 8) | a` matches C
- Output is sorted by key for deterministic/testable output
- No `todo!()` markers remain
- No global state needed (functions operate on `&mut ResourceMap`)

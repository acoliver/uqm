## va_list Handling Strategy

**Chosen Approach:** Internal C helper with Rust coordination

**Architecture:**
1. **Internal C helper** (`uio_vfprintf_helper.c`):
   - Function: `uio_vfprintf_format_helper(format, va_list) -> *char`
   - Uses vsnprintf with buffer resizing strategy
   - Returns malloc'd buffer (caller must free)
   - **NOT an exported uio_* symbol** - compiled into Rust static library via build.rs
   
2. **Rust implementation** (`uio_vfprintf`):
   - Validates arguments (stream, format not NULL)
   - Calls C helper to format va_list into buffer
   - Writes buffer to stream using uio_fwrite
   - Frees buffer using libc::free
   - Returns character count or -1 on error
   - Sets stream status and errno appropriately

3. **C variadic wrapper** (`uio_fprintf`):
   - Implemented in `uio_vfprintf_helper.c`
   - IS an exported uio_* symbol
   - Wraps va_list from variadic args and calls Rust uio_vfprintf
   - Necessary because Rust stable cannot implement variadic functions

**Rationale:**
- Preserves architecture requirement: exported uio_* symbols come from Rust
- Internal helper is implementation detail, not ABI boundary violation
- Pragmatic solution for va_list limitation in stable Rust
- Matches C reference implementation pattern (format → buffer → write)

## Architecture Notes

**Build Boundary Compliance:**
This implementation maintains the architectural requirement that all exported `uio_*` symbols come from Rust:
- [OK] `uio_vfprintf` exported from Rust
- [OK] `uio_fprintf` exported from C (calls Rust uio_vfprintf)
- [OK] `uio_fread` exported from Rust
- [OK] Helper function `uio_vfprintf_format_helper` is internal-only

The C helper is compiled into the Rust static library via build.rs, making it an internal implementation detail rather than a separate build artifact. The `uio_fprintf` symbol, while implemented in C, is justified because:
1. Stable Rust cannot implement variadic functions
2. It's a thin wrapper that delegates to Rust uio_vfprintf
3. Alternative would require nightly Rust or breaking ABI

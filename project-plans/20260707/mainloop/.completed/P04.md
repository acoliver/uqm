## Summary

P04 is a **verification gate**, not an implementation phase. C `main()`
in `sc2/src/uqm.c` owns the full startup sequence (lines 283–452) and
the main-thread event pump (lines 456–472) and subsystem shutdown
(lines 479–507). Rust does **not** replicate, wrap, or call this
sequence. Rust only replaces the body of `Starcon2Main()` (the per-frame
game loop), launched as a separate thread *after* startup completes.

## C Init Sequence (verified against `uqm.c:283-452`)

1. `parseOptions` → `log_init` → `rust_bridge_init` → log file redirect
2. Version/usage/error short-circuits
3. `TFB_PreInit` → `mem_init` → `InitThreadSystem` → `log_initThreads`
   → `initIO` → `prepareConfigDir`
4. `LoadResourceIndex("uqm.cfg")` → `getUserConfigOptions` → global
   options copy → `prepareContentDir`/`prepareMeleeDir`/`prepareSaveDir`/`prepareShadowAddons`
5. `InitTimeSystem` → `InitTaskSystem` → `Alarm_init` → `Callback_init`
   → `Network_init`/`NetManager_init` (NETPLAY)
6. `TFB_InitGraphics` → gamma → `InitColorMaps` → `init_communication`
7. `TFB_SetInputVectors` → `TFB_InitInput`
8. `StartThread(Starcon2Main, ...)` ← **Rust entry boundary**

`initAudio()` is intentionally deferred into `Starcon2Main()` (calls
`AssignTask`, which cannot run on the main thread). Rust inherits this
responsibility.

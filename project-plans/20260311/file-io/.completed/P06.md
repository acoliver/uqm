### 2. Access Mode Checking (REQ-FIO-ACCESS-MODE)

**Changes Made:**
- Implemented proper mode checking in `uio_access()`:
  - `F_OK`: Existence check only
  - `R_OK`: Read permission (always granted for files in mounts)
  - `W_OK`: Write permission (fails on read-only mounts with EACCES)
  - `X_OK`: Execute permission (directories always executable, archives fail, stdio delegates to host)
- Validates mode bits (returns EINVAL for invalid modes)
- Evaluates only the topmost visible object across mount layers
- Properly sets errno for all failure cases

**File Modified:** `rust/src/io/uio_bridge.rs`

## Concurrency Notes

**Mount Registry Iteration/Mutation Contract:**
- Registry uses `Mutex<Vec<MountInfo>>`
- Lock held during snapshot and mutation operations
- `register_mount()` holds lock for entire insertion and sort
- Readers acquire lock for duration of iteration
- No partial insertion/removal states exposed to readers
- Position-based ordering is deterministic and stable

**Thread Safety:**
- All mount operations are atomic under registry lock
- Position assignment is deterministic (no races)
- Read-only flag is set at registration time and immutable
- Multiple concurrent mount operations are serialized by lock

### Implementation Details

#### 1. Helper Functions (Lines 597-660)
Added two helper functions for mount resolution:

**`resolve_mount_for_path<'a>(registry: &'a [MountInfo], virtual_path: &Path) -> Option<MountResolution<'a>>`**
- Finds the topmost mount covering a given virtual path
- Returns `MountResolution` containing mount info and resolved host path
- Uses registry ordering (sorted by position) to respect overlay precedence

**`is_parent_shadowed_by_file(registry: &[MountInfo], virtual_path: &Path, component: &Path) -> bool`**
- Checks if a path component exists as a non-directory file in upper layers
- Used for ENOTDIR parent shadowing detection (prepared for future use)

#### 2. Mount Position Hardening (Lines 535-585)
Fixed mount position assignment to prevent underflow:
- Added `DEFAULT_START_POSITION` constant set to 1000
- Changed all position calculations to use `saturating_sub()` and `saturating_add()`
- TOP insertions now compute `min_position.saturating_sub(1)` instead of `min - 1`
- Updated test helper `add_test_mount` to use same default position scheme

#### 3. Mutation Operations Updates

**`uio_open()` (Lines 2207-2330)**
- **Write modes** (O_WRONLY, O_RDWR, O_CREAT): Resolves through mount registry, checks read_only flag
- **Read mode** (O_RDONLY): Also resolves through mount registry for consistency
- Falls back to direct path resolution when no mount covers the path (backward compatibility)
- Returns EACCES when attempting write on read-only mount
- Returns ENOENT when path doesn't exist

**`uio_rename()` (Lines 134-232)**
- Resolves both old_path and new_path through mount registry
- **Cross-mount detection**: Compares mount IDs, returns EXDEV if different
- **Read-only checks**: Fails with EACCES if either source or destination mount is read-only
- Falls back to direct rename when paths aren't in mounts

**`uio_unlink()` (Lines 2418-2460)**
- Resolves path through mount registry
- Checks read_only flag, returns EACCES on read-only mounts
- Falls back to direct path resolution for non-mounted paths

**`uio_mkdir()` (Lines 370-416)**

- Checks read_only flag, returns EACCES on read-only mounts
- Falls back to direct path resolution for non-mounted paths

**`uio_rmdir()` (Lines 432-478)**

- Falls back to direct path resolution for non-mounted paths

#### 4. Borrow Checker Pattern
All mutation operations use a consistent pattern to avoid E0505 borrow errors:
```rust
let registry = get_mount_registry().lock().unwrap();
let (extracted_data) = if let Some(resolution) = resolve_mount_for_path(&registry, &virtual_path) {
    let data = extract_what_we_need(resolution);
    drop(registry);
    data
} else {
    drop(registry);
    fallback_value
};
```

This pattern extracts needed data (read_only flag, host path) before dropping the registry lock, preventing lifetime conflicts with `MountResolution` which borrows from the registry.

# 1. Gap-to-Code Map

## G1 — Stream status tracking (`uio_feof`, `uio_ferror`, `uio_clearerr`) — CRITICAL
- **Current code location**
  - `rust/src/io/uio_bridge.rs:842-850` — `uio_feof`, `uio_ferror`
  - `rust/src/io/uio_bridge.rs:891-893` — `uio_clearerr`
  - Related stream status writes: `rust/src/io/uio_bridge.rs:1892-1927` (`rust_uio_fread`)
- **Current behavior**
  - `uio_feof()` always returns `1`.
  - `uio_ferror()` always returns `0`.
  - `uio_clearerr()` is a no-op.
  - `rust_uio_fread()` does set `stream.status` to EOF or ERROR on some paths, but the public status accessors ignore that field.
- **Target behavior**
  - Status accessors must reflect actual stream state.
  - EOF set only after a read hits end-of-file.
  - Error set on I/O failure.
  - `uio_clearerr()` clears both flags.
  - `uio_fseek()` / `rewind`-equivalent paths clear EOF per spec.
- **Integration consumers**
  - SDL RWops adapter: `sc2/src/libs/graphics/sdl/sdluio.c`
  - Rust audio stream users via FFI: `rust/src/sound/heart_ffi.rs`
- **Risk if unclosed**
  - EOF vs error misclassification in SDL file adapter.
  - Bad stream semantics for engine/resource/audio reads.
- **REQ IDs**
  - `REQ-FIO-STREAM-STATUS`
  - `REQ-FIO-ERRNO` (secondary when failures occur)
  - `REQ-FIO-ABI-AUDIT` (stream semantics are ABI-visible)

# 6. Edge / Error Handling Map

## 6.1 Path and argument validation
| Area | Detectable invalid cases | Expected behavior |
|---|---|---|
| All pointer-based APIs | `NULL` dir/repo/stream/handle/output-pointer where required | Fail safely; set `errno = EINVAL` where failure is part of API contract |
| Path-taking APIs | invalid C strings / null path pointers / invalid UTF-8 conversion if ABI insists on C string content | Fail safely; no UB; `EINVAL` where detectably invalid |
| `uio_openDir` / `uio_openDirRelative` | unresolved path | `NULL`; `ENOENT` |
| `uio_getFileLocation` | invalid args, null out-params | `-1`; `EINVAL` |
| `uio_getStdioAccess` | invalid args | `NULL`; `EINVAL` |

## 6.2 Path normalization / confinement
| Case | Expected behavior |
|---|---|
| `.` components | remove as no-op |
| `..` components | logical resolution removing previous segment |
| over-traversal above `/` | clamp at `/` |
| repeated `/` | collapse |
| trailing `/` | strip |
| empty path string | refer to handle's own location |
| `..` through mounted subtree | must not escape mount physical root via logical normalization |
| host symlink escape | explicitly not prevented by logical normalization; non-normative limitation |

## 6.3 `uio_access` / access-mode errors
| Case | Expected result |
|---|---|
| `F_OK` on visible entry | `0` |
| `R_OK` on visible entry | `0` |
| `W_OK` on stdio-backed visible object on writable mount | `0` |
| `W_OK` on stdio-backed visible object on read-only mount | `-1`, `EACCES` or `EROFS` |
| `W_OK` on archive-backed object | `-1`, `EACCES` |
| `X_OK` on directory | `0` |
| `X_OK` on stdio-backed file | delegate to host `access()` |
| `X_OK` on archive-backed file | `-1`, `EACCES` |
| invalid/unsupported mode combination | fail safely, `EINVAL` |
| lower writable layer under read-only visible upper layer | do not scan lower layer |

## 6.4 Stream mode/open validation
| Case | Expected result |
|---|---|
| valid mode strings (`r`, `rb`, `w`, `wb`, `a`, `r+`, etc.) | honor stdio-like semantics |
| invalid/unsupported mode string | `NULL`, `errno = EINVAL` |
| opening directory as file | fail, `EISDIR` |
| unsupported open flags in descriptor API | fail, `EINVAL` |
| exclusive create existing path | fail, `EEXIST` |
| truncate on read-only visible file | fail, `EACCES`/`EROFS` |

## 6.5 Stream status/error behavior
| Case | Expected result |
|---|---|
| short read due to EOF | `uio_fread` returns fewer items; EOF flag set; `uio_ferror == 0` |
| read/write I/O failure | error flag set; `uio_ferror != 0` |
| `uio_fseek` after EOF | clears EOF |
| `uio_clearerr` | clears EOF and error |
| `uio_fclose` on valid stream | flush/discard as appropriate; no leak |
| `uio_fclose` on handle invalidated by unmount | still safe for cleanup |

## 6.6 Mutation and overlay errors
| Case | Expected result |
|---|---|
| write existing file where topmost exposing mount is read-only | fail `EACCES`/`EROFS`; no fallthrough |
| create new file absent everywhere with writable covering mount | create there |
| create new file absent everywhere but no writable covering mount | fail `EACCES`/`EROFS` |
| create file where name exists in read-only upper layer | fail `EACCES`/`EROFS` |
| create with `O_EXCL` and visible name exists anywhere | fail `EEXIST` |
| mkdir where visible name already exists | fail `EEXIST` |
| unlink/rmdir visible only in read-only layer | fail `EACCES`/`EROFS`; no whiteouts |
| rename across different backing mounts | fail `EXDEV` |
| parent component shadowed by non-directory | fail `ENOTDIR` |
| parent visible only through read-only layer, no writable same parent path | fail `EACCES`/`EROFS` |

## 6.7 Directory listing errors / edge cases
| Case | Expected result |
|---|---|
| successfully resolved directory, zero matches | non-null empty `uio_DirList` with `numNames == 0` |
| unresolved path | `NULL`, `ENOENT` |
| allocation failure | `NULL`, `ENOMEM` |
| duplicate visible names across mounts | dedup by precedence, one visible name only |
| regex match type advertised | real regex semantics, not heuristics |
| auto-mount listing failure on one entry | if AutoMount implemented, skip failing entry, continue listing, log failure |

## 6.8 FileBlock errors / edge cases
| Case | Expected result |
|---|---|
| invalid handle passed to `uio_openFileBlock[2]` | `NULL`, `EINVAL` |
| offset/size beyond file | clamp/short availability as contract requires, or fail if invalid range; must be defined in implementation slice |
| `uio_accessFileBlock` at EOF | returns available byte count < requested, possibly 0 at EOF; `*buffer` still valid when success path returns 0/short result semantics require |
| repeated `uio_accessFileBlock` | previous internal buffer may be invalidated; returned pointer stable until next access/close |
| `uio_copyFileBlock` read failure | `-1`, `EIO` or mapped underlying errno |
| `uio_clearFileBlockBuffers` | releases cached buffers without invalidating block itself |

## 6.9 Archive mount / archive-read errors
| Case | Expected result |
|---|---|
| corrupt/unreadable central directory at mount | `uio_mountDir` returns `NULL`, `errno = EIO` |
| duplicate entries same normalized path | last central-directory entry wins |
| file entry without explicit directory entries | synthesize implicit directories |
| archive entry path uses `\` or leading `/` | normalize before exposure |
| archive lookups | case-sensitive |
| archive-backed write open | fail `EACCES` |
| archive `uio_access` with `W_OK` or file `X_OK` | fail `EACCES` |
| decompression/CRC failure on read | failing `uio_read` / `uio_fread` returns error status, `errno = EIO` |
| archive entry in union listing | participate in normal dedup/precedence |

## 6.10 `uio_getFileLocation` / stdio-access boundaries
| Case | Expected result |
|---|---|
| stdio-backed file | `uio_getFileLocation` succeeds; `uio_getStdioAccess` direct path |
| stdio-backed single directory | `uio_getFileLocation` succeeds |
| archive-backed file | `uio_getFileLocation` fails `ENOENT`; `uio_getStdioAccess` temp-copy succeeds |
| synthetic archive directory | `uio_getFileLocation` fails `ENOENT` |
| merged multi-mount directory | `uio_getFileLocation` fails `ENOENT` |
| missing file | fail `ENOENT` / `NULL` |
| directory passed to `uio_getStdioAccess` | `NULL`, `EISDIR` |
| cleanup failure during `uio_releaseStdioAccess` temp copy removal | log warning; do not propagate failure |
| direct-path stdio handle after unmount | path may still exist on host, but no longer namespace-guaranteed |
| temp-copy handle after unmount | remains valid until release |

## 6.11 Temp-file / temp-directory handling
| Case | Expected result |
|---|---|
| caller provides `tempDir` | create under caller-provided directory |
| temp name collision | retry with unique name up to reasonable limit |
| cannot establish unique temp path | fail safely |
| temp dir/file cleanup fails | report via logging channel; release still succeeds |
| process-level temp mount init | must exist because P00a resolved required |

## 6.12 Panic-containment fallback map
| Return shape | Required fallback on panic |
|---|---|
| pointer | `NULL` |
| `c_int`, `ssize_t`, `c_long` | `-1` or `EOF` if stdio contract requires |
| `size_t` | `0` |
| void | log + swallow |

Representative functions by family:
- pointer: `uio_openRepository`, `uio_openDir`, `uio_mountDir`, `uio_open`, `uio_fopen`, `uio_getDirList`, `uio_openFileBlock`, `uio_getStdioAccess`

#### [CRITICAL] res_OpenResFile does NOT implement sentinel behavior
- **File**: `rust/src/resource/ffi_bridge.rs`
- **Lines**: 994-1003
- **Evidence**: 
  ```rust

  pub unsafe extern "C" fn res_OpenResFile(
      dir: *mut c_void,
      filename: *const c_char,

  ) -> *mut c_void {
      uio_fopen(dir, filename, mode)
  }
  ```
- **Analysis**: Rust `res_OpenResFile()` simply calls `uio_fopen()` directly - it does NOT:
  1. Call `uio_stat()` to check if path is a directory
  2. Check `S_ISDIR()` macro result
  3. Return `STREAM_SENTINEL` for directories
- **C Reference** (`sc2/src/libs/resource/filecntl.c` lines 28-43):
  ```c
  uio_Stream *
  res_OpenResFile (uio_DirHandle *dir, const char *filename, const char *mode)
  {
      uio_Stream *fp;
      struct stat sb;

      if (uio_stat (dir, filename, &sb) == 0 && S_ISDIR(sb.st_mode))
          return ((uio_Stream *) ~0);  // <-- Sentinel for directory

      fp = uio_fopen (dir, filename, mode);
      return (fp);
  }
  ```
- **Gap**: Rust implementation lacks `uio_stat` extern declaration, so cannot check directory status
- **Status**: DOCUMENTED AS BLOCKER (Phase 08 will add `uio_stat` and implement sentinel logic)

## Structural verification

### No runtime mutex while waiting/writing

`OrderedCommit` uses its own `parking_lot::Mutex<OrderedCommitState>` + `Condvar`.
The `publish_all` method signature is `fn publish_all(&self, sink: &mut dyn Write)`.
No reference to any runtime, state, or external mutex exists in `trace.rs`.
The doc comment on line6 explicitly states: "It never holds the runtime mutex
while waiting or writing." Test `publish_all_does_not_require_runtime_mutex`
verifies this structurally.

### Directory-sync unsupported NOT silently claimed successful

`SyncSupport` is an enum with distinct `Supported` and `Unsupported` variants.
`sync_directory` returns `Ok(SyncSupport::Unsupported)` for EINVAL, `Ok(SyncSupport::Supported)`
for success, and `Err(e)` for all other errors. `DurableResult.dir_sync` carries
the `SyncSupport` value. `IdentityMetadata.dir_sync_supported: bool` records it.
The M10 mutation (classifying ALL errors as Supported) is caught by tests.

### Writer success does not precede drop

`Reservation::commit_record` stores the payload in `self.committed` but does NOT
submit it. Publication happens only in `Reservation::drop` via `inner.submit()`.
The M16 mutation (submitting immediately in `commit_record`) is caught by tests
because the reservation guard is consumed but the record is double-published.

### Existing files are not overwritten

`write_durable` uses `hard_link` for final publication. On Unix, `hard_link`
fails with `AlreadyExists` if the target exists. The collision is handled as an
error (cleanup temp + return error). The M8 mutation (hard_link → rename, which
overwrites) and M9 mutation (skipping the AlreadyExists check) are both caught.

### Tree does not escape root

`TreeManifest::walk_dir` canonicalizes symlinks and checks
`target.starts_with(canonical_root)`. If a symlink target is outside root, it
returns an error. The M15 mutation (disabling the check with `if false`) is
caught. Independent test `tree_rejects_symlink_escape_to_outside` creates a real
symlink to an outside directory and verifies rejection.

## Durability semantics (exact)

The `write_durable` function implements this exact transaction:

1. `fs::create_dir_all(root)` — ensure destination exists
2. `confine_artifact_path(root, label)` — validate label, build confined paths
3. `OpenOptions::new().write(true).create_new(true).open(&temp)` — exclusive temp creation
4. `BufWriter::new(temp_file)` → `write_all(data)` → `flush()` — buffered write + flush
5. `writer.into_inner()` — recover `File` from `BufWriter`
6. `file.sync_all()` — fsync the file data
7. Drop file handle — close
8. `std::fs::hard_link(&temp, &final_path)` — exclusive no-replace publication
   - `AlreadyExists` → error (no overwrite)
   - Other error → error
   - Success → `fs::remove_file(&temp)` — clean up temp
9. `sync_directory(root)` → directory fsync attempt
   - EINVAL → `SyncSupport::Unsupported`
   - Success → `SyncSupport::Supported`
   - Other error → fatal `Err`

Final file is never visible until sync is complete. Temp is cleaned up on all
error paths via `cleanup_temp`.

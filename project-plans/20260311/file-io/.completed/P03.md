## Key Implementation Details

### Stream Status State Machine

```
Initial state: OK
  |
  ├─→ Read 0 bytes at EOF → EOF
  ├─→ Read >0 bytes → OK
  ├─→ Read error → ERROR
  ├─→ Write success → OK
  ├─→ Write error → ERROR
  ├─→ clearerr() → OK
  └─→ Successful seek (if EOF) → OK
```

### Buffer Lifecycle

1. **Allocation**: Stream buffers allocated via `std::alloc::alloc()` are registered in `BUFFER_SIZE_REGISTRY`
2. **Tracking**: Registry maps buffer pointer to size
3. **Deallocation**: `uio_fclose()` looks up size and calls `std::alloc::dealloc()` with proper layout
4. **Safety**: Missing registry entries are treated as external (no deallocation to avoid double-free)

### P02a Carry-Forward Resolution

Per P02a verification override, implemented the following carry-forwards:

1. [OK] **clearerr clears both EOF and error**: `uio_clearerr()` sets status to `UIO_STREAM_STATUS_OK`
2. [OK] **seek/rewind clear EOF**: `uio_fseek()` clears EOF flag on successful seek
3. [OK] **Short write handling**: `uio_fwrite()` checks `write_all()` success for error detection

## Notes

- **Buffer Registry Pattern**: The existing buffer size registry (originally for `uio_DirList`) proved useful for stream buffers as well. This pattern may be applicable to other opaque C structs requiring Rust-side lifecycle management.
- **Status Granularity**: Current implementation uses simple OK/EOF/ERROR states. Future phases may need to distinguish between different error types (e.g., permission denied vs. disk full), but this is sufficient for current requirements.
- **Seek Behavior**: Only SEEK clears EOF; other operations (read/write) only SET status, never clear it (except for `clearerr()`). This matches C stdio semantics.

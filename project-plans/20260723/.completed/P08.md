## Requirement verification

### REQ-TRANSPORT-001: Authenticated transport protocol
- **Version**: `PROTOCOL_VERSION = 1` (u8). Wrong version → `RejectedUnknownCommand`.
- **Nonce**: 256-bit (`[u8; 32]`). Bad nonce → `RejectedBadNonce`.
- **Command IDs**: 3 typed variants (TapDown=1, QuitSmoke=2, Ping=3). Unknown → `RejectedUnknownCommand`.
- **Acks**: 5 types (Accepted, RejectedBadNonce, RejectedReplay, RejectedUnknownCommand, RejectedPushFailed).
- **Replay rejection**: `seen_nonces: HashSet<[u8; 32]>`. Same nonce twice → `RejectedReplay`.
- **Auth order**: version → nonce → replay → command ID → accept (record nonce).
- **Darwin**: `peer_credentials_supported = false` (LOCAL_PEERCRED EINVAL confirmed by P00 probe).
- **No stream credential substitution**: zero stream/listen/accept references.

### REQ-TRANSPORT-002: Path-specific observable counters
- **Counters**: 17 typed `u64` fields covering all 14 required categories from
  `requirements.md`: datagram accept/reject/ack, push/fail, C poll, Rust
  dispatch, post-update, key_observed, quit push/poll/lifecycle, ABI entry,
  active-gate entry, scheduler service, setter writes, replays rejected.
- **is_inactive_accepted**: `active_gate_entry == 0 && scheduler_service == 0 && setter_writes == 0`.
  ABI entry may be nonzero. Verified by 6 dedicated tests.
- **Counter paths distinct**: `counter_paths_are_distinct()` checks all 3 pairwise combinations.

### REQ-TRANSPORT-003: Socket path, mode, packet cap
- `MAX_SOCKET_PATH_LEN = 81` with `check_path_length()` (strict `<`).
- Mode 0600 documented in module docs (production impl sets permissions).
- `PACKETS_PER_PUMP = 16` (compile-time assertions verify >0 and ≤64).

### REQ-PROOF-001: Preflight check
- `PreflightCheck` validates: `fresh_root_created`, `no_matching_processes`, `identity_valid`.
- Pure model — never terminates matching processes (zero kill/signal/exec code).

### REQ-PROOF-002: ChildSession supervision
- State machine: Running → StopRequested → Reaped → PipesClosed → Joined → Complete.
- `record_reap()`: stores exit_code exactly once (state guard: Running|StopRequested only).
  Second reap is no-op (original value preserved, no double-wait).
- `should_kill()`: true only when `SessionState::StopRequested`.
- Kill before reap: failure path is request_stop → should_kill → kill → record_reap.
- `is_reaped()`: true for Reaped and all subsequent states.

### REQ-PROOF-003: Process identity and proof identity
- `ProcessIdentity`: pid (u32), start_time (String), executable_digest (String).
- `matches()`: checks all three fields (pid && start_time && digest).
- `ProofIdentity`: 6 SHA-256 digests (executable, script, content, build, initial_config, final_config).
- `is_valid()`: all 6 must be exactly 64 hex chars (`is_ascii_hexdigit`).

### REQ-PROOF-004: ProofType variants

### REQ-PROOF-005: Hang classification

- ParentHardHang documented: "Child never reached any callback; parent observed hard hang."

### REQ-PROOF-006: Teardown distinctness

- `inactive_teardown_is_distinct()`: inactive != active && inactive != proof && active != proof.

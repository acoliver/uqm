[OK] **Completed**

Implementation:
- **Signature**: Proper 4-parameter signature matching `utils.h`
- **Stdio Mount Handling**: Returns direct path for files on stdio mounts
- **ZIP Mount Handling**: Extracts files to tempDir and returns temp path
- **Error Handling**:
  - Returns NULL with `errno=ENOENT` for missing files
  - Returns NULL with `errno=EISDIR` for directories

- **Path Stability**: C-allocated path string remains valid until handle release

### TASK 3: uio_StdioAccessHandle_getPath(handle)
[OK] **Completed**

Returns the stored `path_cstr` as a const C string pointer. Path remains valid until the handle is released via `uio_releaseStdioAccess`.

### TASK 4: uio_releaseStdioAccess(handle)
[OK] **Completed**

Implementation:
- Frees the C-allocated path string
- For temp copies (`is_temp_copy == true`): deletes temp file (best-effort, logs failures)
- For direct paths: only frees the handle, never deletes underlying file
- Safe to call with NULL handle

### TASK 5: uio_copyFile(srcDir, srcPath, dstDir, dstPath)
[OK] **Completed**

Implementation:
- Opens source via `uio_open` with `O_RDONLY`
- Opens destination via `uio_open` with `O_WRONLY | O_CREAT | O_EXCL` (fails if exists)
- Copies in 8KB chunks using `uio_read`/`uio_write`
- On partial failure: closes handles, removes partial destination via `uio_unlink`, sets `errno=EIO`
- Returns 0 on success, -1 on error

**Related Fix**: Updated `uio_open` to properly handle `O_EXCL` flag by using `create_new(true)` instead of `create(true)` when both `O_CREAT` and `O_EXCL` are set.

## Key Design Decisions

### 1. Path Lifetime Management
Used C-allocated strings (`libc::malloc` via `duplicate_c_string`) for path storage to ensure stability across Rust reallocations. Path pointer remains valid until `uio_releaseStdioAccess` is called.

### 2. Temp Directory Handling
Caller must provide `tempDir` for ZIP-backed files. Function returns `EINVAL` if tempDir is NULL and a ZIP extraction is needed. This puts the responsibility on the caller to manage temp directory cleanup.

### 3. Temp File Cleanup
Cleanup is best-effort with logging. If temp file deletion fails, error is logged but function doesn't return error. This prevents resource cleanup failures from blocking normal operation while still providing visibility.

### 4. O_EXCL Implementation
Fixed `uio_open` to use `OpenOptions::create_new(true)` when both `O_CREAT` and `O_EXCL` flags are set. This ensures atomic create-if-not-exists semantics for `uio_copyFile`.

### 5. Copy Chunk Size
Used 8KB chunks (8192 bytes) for file copying, balancing memory usage and I/O efficiency. This is consistent with typical OS buffer sizes.

## Thread Safety Notes

### StdioAccess Handle Lifetime
- Handles are per-call and independent
- Path strings are C-allocated and stable until release
- No shared state between handles
- Safe across mount topology changes (path is captured at access time)

### Mount Registry Access
- Registry lock is held only during path resolution
- Handle creation occurs outside the lock
- No race conditions between handle lifetime and mount changes
